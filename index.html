What I've Done

  • Real-time Fire Simulation

    • HLSL
    • C#
    • Unity

    (source on GitHub)

    This project was completed as the term project for a computer animation class. As part of the course requirements, I also prepared a conference-style paper (opens in a new tab) describing my research and implementation.

    From the video demo, the first thing you might notice is the big chunky particles. As the focus of this project is on the simulation over high-fidelity graphics, I haven't implemented any sophisticated rendering technique.

    The simulation model I chose (Smoothed Particle Hydrodynamics) is flexible and performant enough to simulate water, fire, or even the formation of stars! The simulation is done using an HLSL compute shader and therefore it runs on the GPU. This drastically increases the number of particles that can be simulated in real-time in order to achieve a visually appealing result.

    This project was made with the game engine Unity to bootstrap development. Here, Unity is simply used for scene management and basic particle rendering. I chose Unity because of my familiarity with the engine.

  • NES Emulator

    • C++
    • Catch2

    (source on GitHub)

    This is a simplified NES emulator. It includes a complete emulator for the 6502 CPU (all opcodes, timing, etc) and a partial implementation of the PPU (no sprite rendering). I created it alongside 3 classmates to fulfill the requirements for a term project.

    To make it possible to complete this project as a class project, we simplified the requirements to only target successfully running a specific test ROM. This ROM simply allows the user to change the screen color by pressing various buttons on the controller.

    While this isn't very visually impressive, it demonstrates the complete CPU emulator, controller input, a simplified PPU (Picture Processing Unit), and cartridge loading.
